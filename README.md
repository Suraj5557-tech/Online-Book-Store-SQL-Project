# Online Book Store - SQL Analytics Project

This repository contains an end-to-end relational database solution for an Online Book Store. The project involves designing the database schema, setting up table relationships, and writing 20 transactional and analytical SQL queries to extract business insights, track sales performance, and optimize inventory management.

---

## 🛠️ Tech Stack & Database Schema
* **Database Management System:** PostgreSQL (pgAdmin 4)
* **Query Language:** SQL (Structured Query Language)
* **Script Editor:** VS Code

The database is built around 3 core tables connected using relational integrity constraints (Primary Keys and Foreign Keys):
1. **CUSTOMERS:** Stores demographic details (ID, Name, Email, Phone, City, Country).
2. **BOOKS:** Manages product catalog (ID, Title, Author, Genre, Price, Stock).
3. **ORDERS:** Captures transactional data (Order ID, Customer ID, Book ID, Order Date, Quantity, Total Amount).

---

## 📊 Business Insights & Problem Solving

The project addresses real-world business requirements through a mix of basic and advanced SQL queries:

### 1. Sales & Revenue Analysis
* **Revenue Tracking:** Calculated the total revenue generated by the store to monitor financial growth.
* **High-Value Orders:** Filtered transactions above $20 to identify premium orders for targeted marketing campaigns.
* **Seasonal Sales:** Isolated order records for specific high-demand months (e.g., November 2023) to evaluate peak sales windows.

### 2. Customer Behavior & Segmentation
* **Geographic Insights:** Segmented data by region to extract country-specific lists (such as identifying all customers based in Canada).
* **Customer Retention:** Utilized `GROUP BY` and `HAVING` clauses to isolate loyal customers who placed **2 or more orders**.
* **VIP Customer Tracking:** Identified the highest-spending customer in the store's transaction history to support customer loyalty rewards.

### 3. Smart Inventory Management (Automated Stock Tracking)
* **Genre Performance:** Analyzed which book categories (e.g., Fantasy, Fiction) drive the highest order volumes.
* **Live Stock Depletion (Query #9):** Implemented a dynamic query using `LEFT JOIN` and `COALESCE` to calculate the remaining stock after order fulfillment:
    
    Remaining Stock = Initial Stock - Total Quantity Sold
    
    *Business Value:* This query automatically flags items nearing zero stock (such as Book ID 184, which dropped from 22 to 13 units) so the store manager can re-order stock before running completely out.

## 🚀 Key SQL Skills Demonstrated
* **Database Design (DDL/DML):** Creating normalized tables, defining data types (`VARCHAR`, `NUMERIC`, `INT`), and enforcing data constraints.
* **Data Handling & Cleaning:** Applying `COALESCE` to handle NULL values cleanly during aggregation tasks.
* **Advanced Joins:** Using `LEFT JOIN` strategically to keep un-ordered books visible in the inventory reports instead of filtering them out.
* **Data Aggregation:** Advanced usage of aggregate functions like `SUM()`, `COUNT()`, `AVG()`, along with data sorting using `GROUP BY`, `ORDER BY`, and `LIMIT`.
